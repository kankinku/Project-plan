
## Documentation Index & Conventions

- **문서 ID**: DOC-000
    
- **제목**: Documentation Index & Conventions
    
- **버전**: 1.0
    
- **목표**: 프로젝트 문서를 일관된 규칙으로 관리하고, 개발/검증/운영에서 “어떤 문서를 어디서 참조해야 하는지”를 단일 인덱스로 제공한다.
    

---

## 1) 문서 분류 체계(Namespace)

- **DOC-***: 문서 운영/규칙/용어/가이드
    
- **SYS-***: 시스템 레벨(전체 구조/데이터/관측/보안/운영)
    
- **PIPE-***: 실행 파이프라인/스테이지 연결(상태 머신, 배치/스트리밍)
    
- **SCHEMA-***: 데이터/전략 DSL/AST/메트릭 스키마(정규화·검증 포함)
    
- **OBS-***: 관측 로그 스펙(결정/주문·체결/트레이드/메타/DQ)
    
- **MOD-***: 모듈별 상세(Interface & Behavior)
    
- **TEST-***: 테스트/검증(단위/통합/회귀/재현성)
    

---

## 2) “진실의 근원(Source of Truth)” 규칙

- 전략 정의(형식/정규화): **SCHEMA-300/310**
    
- 평가 계약(연율화/비용 포함 여부 등): **SCHEMA-340(calc_contract)**
    
- 관측/감사(증거): **OBS-500~540**
    
- 실행 순서/상태: **MOD-400 + PIPE-200**
    
- 엔진 차이(탐색 vs 검증): **MOD-421(A) / MOD-422(B)**
    
- ID/재현성 정책: **MOD-401 + OBS-530**
    

---

## 3) 필수 식별자/버전 필드 규칙

모든 산출물(전략/메트릭/로그/리포트)은 아래를 **반드시 포함**한다.

### 3.1 식별자 키

- `run_id`
    
- `iteration_id`
    
- `strategy_id`
    

### 3.2 버전 키(최소)

- `schema_version` (해당 스키마/로그 버전)
    
- `code.commit_hash` (또는 build_id)
    
- `data.data_version`
    
- `data.integrity_hash`
    
- `engine.version`
    
- `cost_model_version`, `execution_model_version`

## 4) 문서 템플릿 규칙

### 4.1 MOD 문서(단위 기능) — 필수 섹션

- 입력 / 출력 / 상태 / 로그 / 예외 / 테스트
    

### 4.2 SYS 문서(시스템 레벨) — 권장 섹션

- 목표 / 구성요소 / 데이터 흐름 / 운영 정책 / 관측 / 보안 / 테스트·수용 기준
    

### 4.3 SCHEMA/OBS 문서 — 필수 섹션

- 목적 / 필드 정의 / 유효성 규칙 / 예시 JSON / 버전 정책 / 테스트(validator)

## 6) 링크 규칙(문서 간 참조)

- 문서 본문에는 항상 **문서 ID**로 참조한다.
    
    - 예: “재현성 메타데이터는 OBS-530을 따른다.”
        
- 신규 기능 추가 시:
    
    - 관련 **SCHEMA/OBS/MOD/TEST** 링크 없이 머지 금지(권장 운영 규칙)
        

---

## 7) 수용 기준(Acceptance Criteria)

- 모든 run은 **OBS-530 생성**이 필수
    
- backtest 결과는 최소 **SCHEMA-340 + OBS-520**를 포함
    
- 검증(run에서 shortlist 이상)은 **OBS-500/510/520 풀로그**를 기본으로 한다(정책에 따라 예외 가능)
    

---

## 8) 변경 관리(Change Management)

- 스키마/정규화/해시 정책 변경 시:
    
    - 해당 문서의 `버전` 증가
        
    - 영향 범위(기존 결과 재계산 필요 여부) 명시
        
    - 회귀 테스트(TEST-*) 업데이트
